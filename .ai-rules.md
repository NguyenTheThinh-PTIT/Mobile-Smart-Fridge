## 1) Engineering Principles (Global)

- Ưu tiên **correctness > readability > performance > premature optimization**.
- Mọi thay đổi phải có thể maintain bởi team khác sau này.
- Không thêm complexity khi chưa có lý do business/technical rõ ràng.
- Không sửa lan man ngoài phạm vi task.
- Mọi thay đổi phải hướng tới trạng thái **production-ready**.

---

## 2) Architecture & Design (Global)

### 2.1 Backend
- Tuân thủ strict layered architecture:
  - Controller: HTTP boundary, request/response mapping.
  - Service: business logic.
  - Repository/Data access: persistence only.
- Cấm:
  - Business logic trong Controller.
  - DB query trực tiếp trong Controller.
  - Hard-coded config quan trọng.
  - Duplicate logic giữa các service.

### 2.2 Frontend
- Functional components + hooks only.
- Không gọi API trực tiếp trong UI component; dùng service/lib layer.
- Tách state cục bộ và state toàn cục rõ ràng.
- Ưu tiên thiết kế component tái sử dụng.

### 2.3 DevOps
- Docker image phải tối ưu size, có healthcheck, non-root nếu khả thi.
- Compose phải rõ dependency, env management nhất quán, tránh config trùng lặp.

## 3) API Standards (Global)

- RESTful conventions.
- Base URL mặc định: `/api/v1` (nếu project chưa có chuẩn khác).
- HTTP methods:
  - `GET`: read
  - `POST`: create/action
  - `PUT`: replace
  - `PATCH`: partial update
  - `DELETE`: delete

### Unified Response Envelope (bắt buộc nếu không có chuẩn khác)

Success:
```json
{
  "success": true,
  "data": {},
  "error": null
}
```

Error:
```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE_HERE",
    "message": "Mô tả lỗi chi tiết",
    "timestamp": "2026-04-07T20:44:00Z"
  }
}
```

## 4) Coding Standards (Global)

### 4.1 Naming
- Class/Component: PascalCase
- Method/variable: camelCase
- Constant: UPPER_SNAKE_CASE
- DB table/column: snake_case
- API endpoint: kebab-case

### 4.2 Type safety
- TypeScript strict, không dùng `any` trừ khi không tránh được và phải giải thích.
- Java: DTO/request/response rõ ràng, validation rõ ràng.

### 4.3 Error handling
- Backend: centralized exception handling (`@ControllerAdvice`).
- Frontend: lỗi đi qua interceptor/exception mapping thống nhất.

### 4.4 Configuration
- Secrets/config lấy từ env/config file.
- Không hardcode token, client secret, API key trong source code.

## 5) Quality Gates (Global)

Mỗi thay đổi tối thiểu phải đảm bảo:
- Build/compile pass cho scope bị ảnh hưởng.
- Không phá API contract hiện hữu nếu không có migration plan.
- Không tạo warning/error mới rõ ràng trong file đã sửa.

Khuyến nghị:
- Backend: unit test cho service logic quan trọng.
- Frontend: test cho services/hooks quan trọng.

---

## 6) Performance & Reliability (Global)

- Tránh N+1 queries (fetch join/entity graph/projection).
- Danh sách lớn cần pagination/filter.
- Dùng cache khi thật sự có lợi ích rõ ràng.
- Async/background cho tác vụ nặng hoặc IO-bound.
- Tránh blocking calls trong async flow (đặc biệt frontend).

---

## 7) Forbidden Rules (Global)

- Business logic trong Controller/UI glue layer.
- DB access sai layer.
- Duplicate code không cần thiết.
- Hardcoded critical values/secrets.
- Commit debug logs bừa bãi cho production path.
- Thêm dependency nặng khi chưa có lý do rõ ràng.

---

## 8) Generate-Code Checklist (Global)

Khi AI generate code, bắt buộc:
- Tuân thủ rules ở Section 1-7.
- Code compile được trong scope thay đổi.
- Tối ưu tính rõ ràng và khả năng maintain.
- Nếu tạo resource mới, cung cấp đủ lớp/artefacts cần thiết theo kiến trúc dự án.

## 9) Project Profile: Smart Kitchen (Current Project)

### 9.1 Domain requirements
- Fridge item phải có `expirationDate` hợp lệ trong luồng nghiệp vụ.
- Hỗ trợ multi-user theo household.
- Có inventory history/use history.
- Gợi ý công thức dựa trên nguyên liệu sẵn có.
- Realtime sync ưu tiên WebSocket/event-based.

### 9.2 Backend profile (Spring Boot + PostgreSQL)
- Package/feature tổ chức theo module nghiệp vụ hiện hữu trong repo.
- Auth flow ưu tiên JWT access/refresh + OAuth provider integration.
- Flyway migration là nguồn sự thật cho schema/seed thay đổi DB.
- API trả envelope `{success,data,error}` theo Section 3.

### 9.3 Frontend profile (React Native + TypeScript)
- API calls tập trung tại `services`/`features/*/api.ts`.
- Dùng Axios instance + interceptor + exception mapper thống nhất.
- Xử lý đủ loading/empty/error state tại màn hình chính.
- Mobile-first UI, ưu tiên component reusable.

### 9.4 DevOps profile (Docker)
- Tối ưu Docker context (`.dockerignore`) cho từng service.
- Multi-stage build cho backend/ai service khi phù hợp.
- Compose rõ env injection (`env_file` + environment override), healthcheck, depends_on.
